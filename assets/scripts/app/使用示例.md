# Facade + SaveModule 使用示例

本文档说明 `assets/scripts/app` 这套基于 `GameFacade + 子 Facade + SaveModule` 的架构，在外部应该怎么接入。

目标只回答 3 件事：

1. 外部怎么查询数据并显示 UI
2. 外部怎么修改数据
3. 如果新增一个数据，需要补哪些地方

## 一、先理解这套架构的职责

当前主链是：

```text
外部 UI / Scene / Component
    -> GameFacade.instance
        -> 子 Facade（economy / home / progress / shop / gacha / battle）
            -> 业务系统 / Model / 单例
        -> SaveCoordinator
            -> 各个 SaveModule
```

职责分工：

- `GameFacade`
  - 对外统一入口
  - 统一加载存档
  - 统一保存存档
  - 统一暴露各领域 Facade
- `子 Facade`
  - 对外提供“可读、可写”的业务 API
  - 屏蔽底层系统细节
  - 尽量让 UI 不直接依赖底层单例
- `SaveModule`
  - 只关心一个模块自己的存档快照
  - `capture()` 导出
  - `restore()` 恢复
- `SaveCoordinator`
  - 聚合所有模块快照
  - 统一写盘 / 读盘

可以简单理解为：

- 外部读写业务数据，先找 `GameFacade.instance.xxx`
- 外部不要自己直接写本地存储
- 需要持久化的数据，最后都要落到某个 `SaveModule`

## 二、外部如何查询数据来显示 UI

原则：

- UI 只读 `Facade` 暴露的数据
- UI 不要直接摸 `SaveModule`
- UI 不要自己拼接存档结构

### 1. 读取经济数据并显示

适合放在商店、货币栏、背包角标这类 UI。

```ts
import { GameFacade } from '../app/GameFacade';
import { CurrencyType } from '../EconomicSystem/EconomicTypes';

const gold = GameFacade.instance.economy.getCurrencyBalance(CurrencyType.Gold);
const diamond = GameFacade.instance.economy.getCurrencyBalance(CurrencyType.Diamond);
const ticketCount = GameFacade.instance.economy.getItemCount('gacha_ticket');

goldLabel.string = `${gold}`;
diamondLabel.string = `${diamond}`;
ticketLabel.string = `${ticketCount}`;
```

常用入口：

- `GameFacade.instance.economy.getCurrencyBalance(...)`
- `GameFacade.instance.economy.getItemCount(...)`

### 2. 读取 Home 数据并显示

适合首页角色信息、房间状态、角色培养展示。

```ts
import { GameFacade } from '../app/GameFacade';

const homeSnapshot = GameFacade.instance.home.getHomeSnapshot();

levelLabel.string = `${homeSnapshot.level}`;
moneyLabel.string = `${homeSnapshot.money}`;
```

常用入口：

- `GameFacade.instance.home.getHomeSnapshot()`
- `GameFacade.instance.home.collectEquippedRuneIds(heroId)`

### 3. 读取 Progress 数据并显示

适合战斗准备界面、日期选择界面、签到界面、抽卡记录界面。

```ts
import { GameFacade } from '../app/GameFacade';

const progress = GameFacade.instance.progress;
const formation = progress.getFormation();
const heroBuild = progress.getHeroBuild();
const dateSelection = progress.getDateSelection();
const battleProgress = progress.getBattleProgress();

heroIdLabel.string = formation.heroId;
skillCountLabel.string = `${heroBuild.equippedSkillIds.length}`;
dateLabel.string = dateSelection?.dateStr ?? '未选择';
levelLabel.string = battleProgress.activeLevelId ?? '未进入关卡';
```

常用入口：

- `GameFacade.instance.progress.getFormation()`
- `GameFacade.instance.progress.getHeroBuild()`
- `GameFacade.instance.progress.getDateSelection()`
- `GameFacade.instance.progress.getBattleProgress()`
- `GameFacade.instance.progress.getDailyRewardState()`
- `GameFacade.instance.progress.getGachaState()`

### 4. UI 需要实时刷新时怎么做

如果 UI 需要跟着数据变化自动刷新，优先监听对应 `Facade` 的变更事件。

经济域已经有统一监听：

```ts
import { GameFacade } from '../app/GameFacade';
import { CurrencyType } from '../EconomicSystem/EconomicTypes';

private readonly handleEconomyChanged = (): void => {
    const gold = GameFacade.instance.economy.getCurrencyBalance(CurrencyType.Gold);
    this.goldLabel.string = `${gold}`;
};

protected onLoad(): void {
    GameFacade.instance.economy.onChanged(this.handleEconomyChanged);
    this.handleEconomyChanged();
}

protected onDestroy(): void {
    GameFacade.instance.economy.offChanged(this.handleEconomyChanged);
}
```

进度域也支持监听：

```ts
private readonly handleProgressChanged = (): void => {
    const selection = GameFacade.instance.progress.getDateSelection();
    this.dateLabel.string = selection?.dateStr ?? '未选择';
};

protected onLoad(): void {
    GameFacade.instance.progress.onChanged(this.handleProgressChanged);
    this.handleProgressChanged();
}

protected onDestroy(): void {
    GameFacade.instance.progress.offChanged(this.handleProgressChanged);
}
```

`HomeFacade` 当前是 `onHomeChanged/offHomeChanged`：

```ts
private readonly handleHomeChanged = (): void => {
    const snapshot = GameFacade.instance.home.getHomeSnapshot();
    this.levelLabel.string = `${snapshot.level}`;
};

protected onLoad(): void {
    GameFacade.instance.home.onHomeChanged(this.handleHomeChanged);
    this.handleHomeChanged();
}

protected onDestroy(): void {
    GameFacade.instance.home.offHomeChanged(this.handleHomeChanged);
}
```

## 三、外部如何修改数据

原则：

- 外部改数据，优先调 `Facade` 暴露的方法
- 改完后如果希望立即持久化，调用 `GameFacade.instance.saveGame()`
- 不要从 UI 直接改 `SaveModule`
- 不要自己写 `localStorage`

### 1. 修改进度数据

比如保存日期选择：

```ts
import { GameFacade } from '../app/GameFacade';

GameFacade.instance.progress.saveDateSelection({
    text: '测试文本',
    month: 4,
    day: 16,
    dateStr: '4-16',
});

GameFacade.instance.saveGame();
```

比如准备战斗上下文：

```ts
import { GameFacade } from '../app/GameFacade';

const heroId = '000001';

GameFacade.instance.progress.prepareBattle({
    heroId,
    equippedSoulIds: ['soul_001', 'soul_002'],
    equippedSkillIds: [1002, 1004],
    builtinSkillId: null,
    equippedRuneIds: GameFacade.instance.home.collectEquippedRuneIds(heroId),
    enemyIds: ['010001', '010002'],
});

GameFacade.instance.saveGame();
```

### 2. 修改经济数据

比如扣体力、加金币、发道具：

```ts
import { GameFacade } from '../app/GameFacade';
import { CurrencyType } from '../EconomicSystem/EconomicTypes';

GameFacade.instance.economy.spendCurrency(CurrencyType.Stamina, 10, 'start_battle');
GameFacade.instance.economy.addCurrency(CurrencyType.Gold, 200, 'quest_reward');
GameFacade.instance.economy.grantItem('gacha_ticket', 1, 'mail_reward');

GameFacade.instance.saveGame();
```

### 3. 修改商店 / 抽卡 / 战斗相关数据

这类最好调更高层的业务入口，而不是自己拼底层状态。

购买商品：

```ts
const success = GameFacade.instance.shop.buyGoods('general_shop', 'item_001', 1);
if (success) {
    GameFacade.instance.saveGame();
}
```

开始战斗：

```ts
const started = await GameFacade.instance.battle.startBattle({
    levelId: 'level_001',
    heroId: '000001',
    equippedSoulIds: [],
    equippedSkillIds: [1002, 1004],
    builtinSkillId: null,
    equippedRuneIds: GameFacade.instance.home.collectEquippedRuneIds('000001'),
    enemyIds: ['010001', '010001'],
});

if (started) {
    GameFacade.instance.saveGame();
}
```

结论是：

- 外部优先调用 `battle/shop/gacha/progress/economy/home`
- 哪个领域的数据，就走哪个 Facade

## 四、修改数据后，存档是怎么生效的

链路是这样的：

```text
外部调用 Facade 修改数据
    -> 对应 Facade / 系统触发 changed
    -> 对应 SaveModule 被标记 dirty
    -> GameFacade.requestSave()
    -> SaveCoordinator.save()
    -> SaveModule.capture()
    -> 聚合写入本地存储
```

所以外部只需要关心两件事：

1. 业务修改要走 Facade
2. 需要立即落盘时调用 `GameFacade.instance.saveGame()`

## 五、如果新增一个数据，需要填哪些地方

这个问题分两种情况。

### 情况 A：新数据属于已有领域

例如：

- 新增一个签到字段，属于 `ProgressFacade`
- 新增一个货币字段，属于 `EconomyFacade`
- 新增一个 Home 展示状态，属于 `HomeFacade`

这种情况通常不用新建 `SaveModule`，而是补齐已有 Facade 和已有 SaveModule 依赖的数据导出。

#### 需要补的地方

1. 先确定数据归属

- 经济相关：放 `EconomyFacade`
- Home / 角色 / 家园相关：放 `HomeFacade`
- 进度 / 战斗准备 / 日期 / 签到 / 抽卡记录相关：放 `ProgressFacade`

2. 在对应领域的运行时数据结构里新增字段

例如在 `ProgressFacade` 的保存结构里新增：

```ts
export interface IProgressSaveData {
    formation: IPlayerFormation;
    heroBuild: IHeroBuild;
    runes: IRuneLoadout;
    dateSelection: IDateSelectionSaveData | null;
    battleContext: IBattleContext;
    dailyReward?: IDailyRewardState;
    gacha?: IGachaSaveData;
    battle?: IBattleProgressState;
    yourNewField?: IYourNewField;
}
```

3. 在 Facade 内新增字段和读写 API

以 `ProgressFacade` 为例：

```ts
private yourNewField: IYourNewField = {
    value: 0,
};

public getYourNewField(): IYourNewField {
    return { ...this.yourNewField };
}

public setYourNewField(value: IYourNewField): void {
    this.yourNewField = { ...value };
    this.notifyChanged();
}
```

4. 在导出方法里补上

```ts
public exportSave(): IProgressSaveData {
    return {
        ...,
        yourNewField: { ...this.yourNewField },
    };
}
```

5. 在导入方法里补上

```ts
public importSave(snapshot: Partial<IProgressSaveData> | null | undefined): void {
    ...
    this.yourNewField = snapshot.yourNewField
        ? { ...snapshot.yourNewField }
        : { value: 0 };
    this.notifyChanged();
}
```

6. 如果 UI 要使用，就通过 Facade 新 API 读取

```ts
const data = GameFacade.instance.progress.getYourNewField();
```

#### 这一类的最小清单

- 新字段类型定义
- Facade 内部状态字段
- Facade 的 `get/set`
- `exportSave()`
- `importSave()`
- 修改时触发 `notifyChanged()` 或对应变更事件
- UI 改为从 Facade 读

### 情况 B：新数据是一个独立领域

例如：

- 新增邮件系统
- 新增任务系统
- 新增成就系统

这种情况建议新增独立 Facade 和独立 SaveModule。

#### 需要补的地方

1. 新建一个 Facade

例如：

- `assets/scripts/app/facades/MailFacade.ts`

职责：

- 对外提供读 API
- 对外提供改 API
- 管理自己的运行时状态
- 提供变更通知
- 提供导出 / 导入存档快照的方法

示意：

```ts
export interface IMailSaveData {
    mails: IMailItem[];
}

export class MailFacade {
    private readonly listeners = new Set<() => void>();
    private mails: IMailItem[] = [];

    public getMails(): IMailItem[] {
        return this.mails.map((mail) => ({ ...mail }));
    }

    public addMail(mail: IMailItem): void {
        this.mails = [...this.mails, { ...mail }];
        this.notifyChanged();
    }

    public exportSave(): IMailSaveData {
        return {
            mails: this.mails.map((mail) => ({ ...mail })),
        };
    }

    public importSave(snapshot: IMailSaveData | null | undefined): void {
        this.mails = snapshot?.mails?.map((mail) => ({ ...mail })) ?? [];
        this.notifyChanged();
    }

    public onChanged(listener: () => void): void {
        this.listeners.add(listener);
    }

    public offChanged(listener: () => void): void {
        this.listeners.delete(listener);
    }

    private notifyChanged(): void {
        this.listeners.forEach((listener) => listener());
    }
}
```

2. 新建一个 SaveModule

例如：

- `assets/scripts/app/save/modules/MailSaveModule.ts`

示意：

```ts
import { DirtySaveModule } from '../../../save/modules/DirtySaveModule';
import { MailFacade, type IMailSaveData } from '../../facades/MailFacade';

export class MailSaveModule extends DirtySaveModule<IMailSaveData> {
    public readonly key = 'mail';

    constructor(private readonly mailFacade: MailFacade) {
        super();
        this.mailFacade.onChanged(this.handleChanged);
    }

    public capture(): IMailSaveData {
        return this.mailFacade.exportSave();
    }

    public restore(snapshot: IMailSaveData | null | undefined): void {
        this.mailFacade.importSave(snapshot);
    }

    private readonly handleChanged = (): void => {
        this.markDirty();
    };
}
```

3. 在 `GameFacade` 里挂出这个 Facade

需要补：

- import 新 Facade
- import 新 SaveModule
- 新增公开字段
- 在构造函数里实例化
- 在 `registerSaveModules()` 里注册

示意：

```ts
public readonly mail: MailFacade;

private constructor() {
    ...
    this.mail = new MailFacade();
}

private registerSaveModules(): void {
    ...
    this.saveCoordinator.register(new MailSaveModule(this.mail));
}
```

4. 外部通过 `GameFacade.instance.mail` 使用

```ts
const mails = GameFacade.instance.mail.getMails();
GameFacade.instance.mail.addMail(newMail);
GameFacade.instance.saveGame();
```

#### 这一类的完整清单

- 新的保存数据接口
- 新的 Facade
- Facade 的读 API
- Facade 的写 API
- Facade 的变更通知
- Facade 的 `exportSave()`
- Facade 的 `importSave()`
- 新的 `SaveModule`
- `GameFacade` 挂载和注册
- 外部 UI 改为通过 `GameFacade.instance.新领域` 访问

## 六、推荐接入顺序

以后新增数据时，推荐按这个顺序判断：

1. 这个数据属于哪个业务域
2. 先决定是放到已有 Facade，还是单独新建 Facade
3. 先把“运行时读写 API”补出来
4. 再补“导出 / 导入存档”
5. 再补 `SaveModule`
6. 最后让 UI 改为只通过 `GameFacade` 访问

## 七、不推荐的写法

下面这些尽量不要做：

- UI 直接读写 `GameContext.instance`
- UI 直接操作 `Economy`、`HomeStatusModel`、`SoulSystem`、`RuneSystem`
- UI 直接操作 `SaveModule`
- UI 自己写 `localStorage`
- 把所有新业务都继续堆到 `GameFacade` 本体

## 八、可以直接套用的思路

你可以把这套架构理解成一句话：

```text
外部只管找 GameFacade -> 领域 Facade；
领域 Facade 只管业务读写；
SaveModule 只管持久化；
SaveCoordinator 只管聚合存档。
```

如果后面你要继续扩字段，优先看这个顺序：

```text
先定领域
    -> 再补 Facade 的 get/set
        -> 再补 export/import
            -> 再决定是否需要新 SaveModule
```
