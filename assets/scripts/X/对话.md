# 对话系统节点结构与快进优化示例

本文档基于当前 `DialogView.ts`、`DialogConfig.ts` 的实现整理，包含：

- 对话节点结构示例
- 分支选项样式配置示例
- 快进相关优化方案
- `DialogView.ts` 关键接入代码

## 1. 基础节点结构

当前 `DialogConfig.ts` 中的核心结构可以理解为：

```ts
export interface DialogChoice {
    text: string;
    targetDialogId: string;
    targetLineIndex?: number;
    condition?: string;
    textColor?: string;
    bgPath?: string;
}

export interface DialogLine {
    name: string;
    text: string;
    avatarPath?: string;
    isFlashback?: boolean;
    voicePath?: string;
    sfxPath?: string;
    nextDialogId?: string;
    choices?: DialogChoice[];
    params?: Record<string, string>;
}
```

说明：

- `textColor` 用于分支按钮文字颜色，格式如 `"#FFFFFF"`
- `bgPath` 用于分支按钮底图路径，可走 bundle 资源或远程资源加载

如果要接入本文档中的表现层扩展，建议在业务层继续扩展：

```ts
type DialogVisualEffect
    = { type: 'shakeMask'; duration?: number; intensity?: number; count?: number }
    | { type: 'bounceAvatar'; duration?: number; height?: number; count?: number }
    | { type: 'pulseAvatar'; duration?: number; scale?: number };

export interface DialogLineEx extends DialogLine {
    backgroundPath?: string;
    affectionChanges?: Record<string, number>;
    visualEffects?: DialogVisualEffect[];
}
```

## 4. `DialogView.ts` 关键接入点

### 4.1 打字机拦截

快进时不再开启逐字定时器，直接显示整句文本并触发完成回调。

```ts
private startTypewriter(text: string, onComplete?: () => void, instant = false) {
    this._fullText = text;
    this._shownLength = 0;
    this._isTyping = true;
    this._typewriterComplete = onComplete ?? null;

    if (this.contentLabel) {
        this.contentLabel.string = '';
    }

    this.unschedule(this.updateTypewriter);

    if (this.isSkipping || instant) {
        this.finishTypingImmediately();
        return;
    }

    this.schedule(this.updateTypewriter, this._normalTypewriterInterval);
}
```

### 4.2 音频拦截

快进时不播放语音和打字音效。

```ts
if (line.voicePath && !this.isSkipping) {
    audioManager.instance.playSound(line.voicePath);
}

if (line.sfxPath && !this.isSkipping) {
    audioManager.instance.playSound(line.sfxPath);
}
```

### 4.3 立绘 / 背景防抖

快进时把视觉资源切换推迟 50ms。如果 50ms 内进入下一句，则取消上次请求，只保留最后一次。

```ts
private readonly _skipVisualDebounceDelay = 0.05;
private _pendingVisualLine: DialogVisualLine | null = null;

private updateVisualsWithDebounce(line: DialogVisualLine): void {
    if (!this.isSkipping) {
        this.cancelPendingVisualUpdate();
        this.renderVisualsImmediately(line);
        return;
    }

    this._pendingVisualLine = line;
    this.unschedule(this.flushPendingVisualUpdate);
    this.scheduleOnce(this.flushPendingVisualUpdate, this._skipVisualDebounceDelay);
}

private cancelPendingVisualUpdate(): void {
    this._pendingVisualLine = null;
    this.unschedule(this.flushPendingVisualUpdate);
}

private readonly flushPendingVisualUpdate = () => {
    const line = this._pendingVisualLine;
    this._pendingVisualLine = null;

    if (!line) {
        return;
    }

    this.renderVisualsImmediately(line);
};
```

### 4.4 数据效果与表现特效分离

原则：

- 数据逻辑始终执行
- 视觉特效在快进时直接跳过
- 即使跳过动画，也必须手动补发 `onComplete`

```ts
private applyLineDataEffects(line: DialogEffectLine): void {
    if (!line.affectionChanges) {
        return;
    }

    for (const [characterId, delta] of Object.entries(line.affectionChanges)) {
        if (!Number.isFinite(delta) || delta === 0) {
            continue;
        }

        AffectionManager.instance.changeAffection(characterId, delta, this._currentDialogId ?? undefined);
    }
}

private playVisualEffectSafely(effect: DialogVisualEffect, onComplete?: () => void): void {
    if (this.isSkipping) {
        onComplete?.();
        return;
    }

    switch (effect.type) {
        case 'shakeMask':
            if (!this.maskNode) {
                onComplete?.();
                return;
            }
            UIAnimation.shake(this.maskNode, effect.intensity ?? 10, effect.count ?? 3, {
                duration: effect.duration ?? 0.4,
                onComplete,
            });
            return;

        case 'bounceAvatar':
            if (!this.avatarSprite?.node) {
                onComplete?.();
                return;
            }
            UIAnimation.bounce(this.avatarSprite.node, effect.height ?? 20, effect.count ?? 3, {
                duration: effect.duration ?? 0.5,
                onComplete,
            });
            return;

        default:
            onComplete?.();
            return;
    }
}
```

### 4.5 分支按钮动态样式注入

分支按钮仍然使用 `choiceButtonPrefab` 作为基础样式，但允许在配置里动态覆盖：

- `choice.textColor` 控制文字颜色
- `choice.bgPath` 控制按钮底图

```ts
private showChoices(choices: DialogChoice[]) {
    if (!choices || choices.length === 0) {
        return;
    }

    this.isSkipping = false;
    this.updateSkipButtonState();
    this.unschedule(this.advanceAfterSkip);
    this.renderCurrentVisualsImmediately();

    if (this.choiceContainer) {
        this.choiceContainer.removeAllChildren();
        choices.forEach((choice) => {
            if (choice.condition && this._conditionEvaluator && !this._conditionEvaluator(choice.condition)) {
                return;
            }

            if (this.choiceButtonPrefab) {
                const node = instantiate(this.choiceButtonPrefab) as Node;
                const label = node.getComponentInChildren(Label);
                if (label) {
                    label.string = choice.text;

                    if (choice.textColor) {
                        try {
                            const color = new Color().fromHEX(choice.textColor);
                            label.color = color;
                        } catch (err) {
                            console.warn('[DialogView] Invalid choice textColor:', choice.textColor, choice.text);
                        }
                    }
                }

                const bgSprite = node.getComponent(Sprite);
                if (bgSprite && choice.bgPath) {
                    resourceUtil.setSpriteFrame(choice.bgPath, bgSprite, (err) => {
                        if (err) {
                            console.warn('[DialogView] Choice background load failed:', choice.bgPath, choice.text, err);
                        }
                    });
                } else if (!bgSprite && choice.bgPath) {
                    console.warn('[DialogView] Choice button Sprite missing for bgPath:', choice.bgPath, choice.text);
                }

                node.on(EventType.TOUCH_END, () => this.onChoiceSelected(choice), this);
                this.choiceContainer.addChild(node);
            } else {
                console.warn('[DialogView] ChoiceButtonPrefab not set:', choice.text);
            }
        });
    } else {
        console.warn('[DialogView] ChoiceContainer not bound:', choices.map((item) => item.text).join(', '));
    }
}
```

使用建议：

- prefab 提供默认按钮样式
- 配置字段只做覆盖，不写时继续沿用 prefab 默认值
- `bgPath` 推荐统一用同一种资源协议，避免线上配置混杂

## 5. 快进单帧分片架构

目标：

- 不再对每一句已读对白都单独 `scheduleOnce`
- 在同一帧内尽可能多处理几个已读节点
- 遇到未读、分支、预算超时再让出控制权

```ts
private readonly _skipFrameBudgetMs = 8;
private _skipSliceRunning = false;
private _skipSliceScheduled = false;

private nowMs(): number {
    return globalThis.performance?.now?.() ?? Date.now();
}

private onCurrentLineSettled(line: DialogEffectLine): void {
    if (line.choices && line.choices.length > 0) {
        this.isSkipping = false;
        this.updateSkipButtonState();
        this.showChoices(line.choices);
        return;
    }

    if (this.isSkipping) {
        this.requestSkipSlice();
        return;
    }

    if (this._autoPlay) {
        this.scheduleOnce(this.advanceAfterAutoPlay, this._autoInterval);
    }
}

private requestSkipSlice(): void {
    if (!this.isSkipping || this._choiceActive || this._skipSliceRunning) {
        return;
    }

    this.processSkipSlice();
}

private queueSkipSliceNextFrame(): void {
    if (!this.isSkipping || this._choiceActive || this._skipSliceScheduled) {
        return;
    }

    this._skipSliceScheduled = true;
    this.scheduleOnce(this.runQueuedSkipSlice, 0);
}

private readonly runQueuedSkipSlice = () => {
    this._skipSliceScheduled = false;
    this.processSkipSlice();
};

private processSkipSlice(): void {
    if (!this.isSkipping || this._choiceActive || this._skipSliceRunning) {
        return;
    }

    this._skipSliceRunning = true;

    try {
        const frameStart = this.nowMs();

        while (this.isSkipping && !this._choiceActive) {
            const result = this.advanceOneSkipNode();

            if (result !== 'continued') {
                return;
            }

            if (this.nowMs() - frameStart >= this._skipFrameBudgetMs) {
                this.queueSkipSliceNextFrame();
                return;
            }
        }
    } finally {
        this._skipSliceRunning = false;
    }
}

private advanceOneSkipNode(): 'continued' | 'stopped' | 'finished' {
    const currentLine = this._lines[this._index] as DialogEffectLine | undefined;
    if (!currentLine) {
        this.finishDialog();
        return 'finished';
    }

    const nextIndex = this._index + 1;
    if (nextIndex >= this._lines.length) {
        this.finishDialog();
        return 'finished';
    }

    const nextLine = this._lines[nextIndex] as DialogEffectLine;

    if (!this.canFastForwardLine(nextIndex)) {
        this.isSkipping = false;
        this.updateSkipButtonState();
        this._index = nextIndex;
        this.showCurrentLine();
        return 'stopped';
    }

    this._index = nextIndex;
    this.renderLineImmediatelyForSkip(nextLine);

    if (nextLine.choices && nextLine.choices.length > 0) {
        this.isSkipping = false;
        this.updateSkipButtonState();
        this.showChoices(nextLine.choices);
        return 'stopped';
    }

    return 'continued';
}
```

## 6. 使用建议

- `DialogLine` 基础字段继续保留在 `DialogConfig.ts`，不要直接破坏现有兼容性
- `backgroundPath`、`affectionChanges`、`visualEffects` 可以作为业务扩展字段逐步接入
- `textColor`、`bgPath` 属于分支按钮表现层字段，适合做 Data-Driven UI 配置
- 快进时所有高频表现逻辑都应该满足同一原则：
  - 音频直接拦截
  - 打字机直接完成
  - 视觉资源延迟合并
  - 特效跳过但补发完成回调
  - 已读推进使用单帧分片而不是每句一个定时器

## 8. 对话预制体节点结构推荐

下面是一个适合当前 `DialogView.ts` 的推荐结构。目标是让脚本字段一一对应，避免运行期再做层级查找。

### 8.1 主对话预制体结构

```txt
DialogRoot
├─ Mask
│  └─ Panel
│     ├─ Background
│     │  └─ BgSprite
│     ├─ Character
│     │  └─ AvatarSprite
│     ├─ Header
│     │  └─ NameLabel
│     ├─ ContentArea
│     │  ├─ ContentClickArea
│     │  └─ ContentLabel
│     ├─ ChoiceArea
│     │  └─ ChoiceContainer
│     └─ Controls
│        └─ SkipButton
```

推荐说明：

- `DialogRoot`：挂 `DialogView` 脚本
- `Mask`：整屏点击遮罩，建议带 `UIOpacity`
- `BgSprite`：对白背景图或场景底图节点，挂 `Sprite`
- `AvatarSprite`：角色立绘节点，挂 `Sprite`
- `NameLabel`：角色名，挂 `Label`
- `ContentClickArea`：正文点击区域，用来接收继续点击
- `ContentLabel`：对白正文，挂 `Label`
- `ChoiceContainer`：分支按钮父节点，通常挂 `Layout`
- `SkipButton`：跳过按钮，节点上挂 `Button`

### 8.2 分支按钮 prefab 结构

当前 `showChoices()` 的逻辑是：

- 根节点 `instantiate(this.choiceButtonPrefab)`
- 根节点上取 `Sprite` 作为按钮底图
- 子节点里取 `Label` 作为按钮文字

所以推荐 `choiceButtonPrefab` 结构如下：

```txt
ChoiceButton
├─ Label
```

组件建议：

- `ChoiceButton`
  - `Button`
  - `Sprite`
  - 可选 `UIOpacity`
- `Label`
  - `Label`

如果后续要做更复杂的分支按钮，比如图标、角标、锁定态，也建议继续保持：

- 根节点仍然有 `Sprite`
- 子树中至少有一个可被 `getComponentInChildren(Label)` 找到的 `Label`

### 8.5 绑定完成后的运行逻辑

主对话预制体在运行时主要依赖以下几段逻辑：

1. `showCurrentLine()` 更新名字、正文、立绘、背景、音频和特效
2. `showChoices()` 动态实例化 `choiceButtonPrefab`
3. `toggleSkip()` 控制快进状态
4. `onClickNext()` 负责点击推进

分支按钮的动态样式依赖以下结构约定：

```ts
const node = instantiate(this.choiceButtonPrefab) as Node;
const label = node.getComponentInChildren(Label);
const bgSprite = node.getComponent(Sprite);
```

所以如果 prefab 改层级，必须保证：

- 根节点仍能拿到 `Sprite`
- 子节点树里仍能拿到 `Label`

### 8.6 推荐的编辑器检查清单

- `DialogRoot` 是否挂了 `DialogView`
- `maskNode`、`contentClickArea` 是否都已绑定
- `contentLabel` 是否是实际显示正文的 `Label`
- `avatarSprite`、`backgroundSprite` 是否挂了 `Sprite`
- `choiceContainer` 是否为空节点且允许动态添加子节点
- `choiceButtonPrefab` 根节点是否有 `Sprite`
- `choiceButtonPrefab` 子节点是否有 `Label`
- `skipButton` 是否绑定了真正的 `Button`

如果这几项都满足，当前文档中的代码可以直接工作。
