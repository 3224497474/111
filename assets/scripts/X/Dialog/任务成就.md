# 任务 & 成就系统使用说明（任务成就.md）

本文档说明 X 框架下任务 / 成就系统的整体结构、使用方式和完整示例，方便在当前项目和后续项目复用。

---

## 1. 模块总览

主要文件（均在 `assets/scripts/X/client/Script/easyFramework/mgr`）：

- `TaskTypes.ts`
  - 定义任务枚举：`TaskType`、`TaskState`
  - 定义任务配置和运行时结构：`TaskConfig`、`TaskRuntimeState`
  - 定义成就配置和运行时结构：`AchievementConfig`、`AchievementRuntimeState`
- `TaskExpression.ts`
  - 简单条件表达式解析工具 `TaskExpression`
  - 支持形如 `kill_monster>=10 && level>=5` 的字符串表达式
- `ProgressMetrics.ts`
  - 公共“计数中心” `ProgressMetrics`
  - 用来存各种用于任务/成就的统计数值（击杀数、通关数、登录次数等）
- `TaskManager.ts`
  - 任务系统核心逻辑 `TaskManager` / `taskManager`
  - 管理任务状态流转：`Locked → Available → Accepted → CanReward → Completed / Failed`
  - 负责任务的本地存储（`GameStorage`）
  - 支持“每日任务重置”
- `AchievementManager.ts`
  - 成就系统核心逻辑 `AchievementManager` / `achievementManager`
  - 自动根据 `ProgressMetrics` 评估成就是否达成
  - 管理“可领奖 / 已完成”状态，并存储到本地

辅助文件：

- `assets/scripts/X/client/Script/config/TaskConfigData.ts`
  - 示例任务配置数组：`TASK_CONFIGS`
  - 示例成就配置数组：`ACHIEVEMENT_CONFIGS`
- `assets/scripts/X/client/Script/easyFramework/mgr/TaskSystemBridge.ts`
  - 桥接组件 `TaskSystemBridge`
  - 将 `GD.event` / `Notifications` 与 TaskManager / AchievementManager / ProgressMetrics 串起来
  - 处理跨天事件 `GD.event.overDay`，用于每日任务重置

## 2. 任务配置：TaskConfig

### 2.1 配置结构

`TaskConfig` 主要字段（见 `TaskTypes.ts`）：

```ts
export interface TaskConfig {
    id: number;                  // 任务 ID
    type: TaskType;              // 任务类型（Main/Daily/OneShot/LimitTime）

    titleKey: string;            // 标题多语言 key
    descKey: string;             // 描述多语言 key
    rewardTextKey?: string;      // 奖励说明多语言 key（可选）

    groupKey?: string;           // UI 分组用 key（例如 'main' / 'daily'）
    sortOrder?: number;          // 排序权重，越小越靠前

    unlockExpr?: string;         // 解锁条件表达式（可选）
    completeExpr: string;        // 完成条件表达式（必填）

    progressMetricKey?: string;  // 进度展示用的 metric key（可选）
    progressTarget?: number;     // 进度目标值（可选）

    preTaskId?: number;          // 前置任务 ID（任务链）
    startTime?: number;          // 限时任务开始时间（秒）
    endTime?: number;            // 限时任务结束时间（秒）
    isHiddenBeforeUnlock?: boolean; // 未解锁时是否在列表中隐藏
}
```

任务类型枚举：

```ts
export enum TaskType {
    Main = 1,      // 主线 / 引导任务
    Daily = 2,     // 每日任务
    OneShot = 3,   // 一次性成长 / 支线任务
    LimitTime = 4, // 限时活动任务
}
```

任务状态枚举：

```ts
export enum TaskState {
    Locked = 0,    // 未解锁
    Available = 1, // 可接
    Accepted = 2,  // 已接受 / 进行中
    CanReward = 3, // 已达成，可领奖
    Completed = 4, // 已领奖，完成
    Failed = 5,    // 失败（超时等）
}
```

## 4. 条件表达式语法（TaskExpression）

文件：`TaskExpression.ts`

支持简单表达式：

- 原子条件：`metricName op number`
  - `op` 支持：`>`, `>=`, `<`, `<=`, `==`, `!=`
  - 例如：`kill_monster_daily>=10`、`player_level>=5`
- 逻辑连接：
  - `&&`：AND
  - `||`：OR
- 限制：
  - 不支持括号 `()`；
  - 不支持同时混用 `&&` 和 `||`。

示例：

```ts
unlockExpr   = 'player_level>=5';
completeExpr = 'kill_monster_daily>=10 && login_today>=1';
completeExpr = 'kill_monster_total>=100 || gold_total>=50000';
```

系统内部评估：

```ts
TaskExpression.evaluate(expr, (key) => ProgressMetrics.instance.get(key));
```

## 5. 进度计数：ProgressMetrics

文件：`ProgressMetrics.ts`

核心 API：

```ts
ProgressMetrics.instance.update(metricKey: string, delta: number): number;
ProgressMetrics.instance.set(metricKey: string, value: number): void;
ProgressMetrics.instance.get(metricKey: string): number;
```

用法示例：

```ts
// 击杀怪物一次
ProgressMetrics.instance.update('kill_monster_total', 1);

// 今日击杀计数
ProgressMetrics.instance.update('kill_monster_daily', 1);

// 设置玩家等级
ProgressMetrics.instance.set('player_level', 5);
```

任务和成就的 `completeExpr` 中使用的 `metricKey`，必须和这里保持一致。

## 6. 初始化任务 / 成就系统

### 6.1 通过 TaskSystemBridge 自动初始化

文件：`TaskSystemBridge.ts`，挂在一个常驻节点上。

```ts
@ccclass('TaskSystemBridge')
export class TaskSystemBridge extends Component {

    onLoad() {
        // 初始化任务 & 成就配置
        taskManager.init(TASK_CONFIGS);
        achievementManager.init(ACHIEVEMENT_CONFIGS);

        // 监听跨天事件：日常任务重置
        Notifications.on(GD.event.overDay, this.onOverDay, this);

        // 示例：游戏开始视为“今日登录一次”
        // Notifications.on(GD.event.gameStart, this.onGameStart, this);

        // 示例：击杀怪物事件
        // Notifications.on(GD.event.killMonster, this.onKillMonster, this);
    }

    onDestroy() {
        Notifications.off(GD.event.overDay, this);
        // Notifications.off(GD.event.gameStart, this);
        // Notifications.off(GD.event.killMonster, this);
    }

    private onOverDay() {
        taskManager.resetDailyTasks();

        ProgressMetrics.instance.set('login_today', 0);
        ProgressMetrics.instance.set('kill_monster_daily', 0);

        taskManager.evaluateAll();
        achievementManager.handleMetricsUpdated();
    }

    private onGameStart() {
        ProgressMetrics.instance.update('login_today', 1);
        taskManager.evaluateAll();
        achievementManager.handleMetricsUpdated();
    }

    private onKillMonster(params: { monsterId: number }) {
        ProgressMetrics.instance.update('kill_monster_total', 1);
        ProgressMetrics.instance.update('kill_monster_daily', 1);

        taskManager.evaluateAll();
        achievementManager.handleMetricsUpdated();
    }
}
```

**挂载方式：**

- 在 Home 场景或 GameRoot 场景的根节点上添加 `TaskSystemBridge` 组件；
- 确保 `GD.event.overDay` 在 X 框架中已经根据时间逻辑正确触发。

## 7. 任务系统使用示例（UI）

### 7.1 获取任务列表

```ts
import { taskManager } from '../easyFramework/mgr/TaskManager';
import { TaskType, TaskState } from '../easyFramework/mgr/TaskTypes';

// 获取所有任务
const allTasks = taskManager.getAllTasks();

// 获取每日任务
const dailyTasks = taskManager.getTasksByType(TaskType.Daily);

for (const t of dailyTasks) {
    const cfg = t.config;
    const state = t.state;
    // 根据 cfg.titleKey / cfg.descKey 去文案表取展示文本
}
```

### 7.2 接受任务 / 领取奖励

```ts
// 接受任务（在“可接任务”列表中点击“接取”）
taskManager.acceptTask(taskId);

// 领取任务奖励（在“可领奖”状态点击“领取”）
taskManager.claimReward(taskId);
```

UI 层可根据 `TaskState`：

- `Available`：显示“接取”按钮；
- `Accepted`：显示进度条 / “前往”按钮；
- `CanReward`：显示“领取奖励”按钮 + 红点；
- `Completed`：显示“已完成”；
- `Failed`：显示“已失败”（可选，根据项目需要决定是否展示）。

## 9. 红点逻辑示例

### 9.1 主入口红点（任务）

```ts
import { TaskType } from '../easyFramework/mgr/TaskTypes';
import { taskManager } from '../easyFramework/mgr/TaskManager';

// 任意任务可领奖
const hasAnyReward = taskManager.hasRewardableTask();

// 仅每日任务是否有可领
const hasDailyReward = taskManager.hasRewardableTask(TaskType.Daily);
```

### 9.2 主入口红点（成就）

```ts
import { achievementManager } from '../easyFramework/mgr/AchievementManager';

const hasAchReward = achievementManager.hasRewardable();

const showMainRedPoint =
    taskManager.hasRewardableTask() || achievementManager.hasRewardable();
```

UI 中将红点节点 `redPointNode.active = showMainRedPoint;` 即可。

## 11. 数据存储说明

- 使用现有 `GameStorage` / `GameMemoryStorage`；
- 键名约定：
  - 任务状态：`TaskState_<taskId>`
  - 任务进度：`TaskProgress_<taskId>`
  - 成就完成：`AchieveCompleted_<achId>`
  - 成就可领奖：`AchieveCanReward_<achId>`

调试或清档时，可以通过 `GameStorage` 清除这些键，或直接调用 `GameStorage.clear()`。

## 12. 集成步骤小结

1. 确认以下文件存在并通过编译：
   - `TaskTypes.ts`、`TaskExpression.ts`、`ProgressMetrics.ts`、`TaskManager.ts`、`AchievementManager.ts`
   - `config/TaskConfigData.ts`
   - `mgr/TaskSystemBridge.ts`
2. 在常驻场景节点上挂 `TaskSystemBridge`，让其在 `onLoad` 中初始化任务 / 成就系统。
3. 在核心玩法逻辑中，调用 `ProgressMetrics.instance.update/set` 更新计数，再调用
   `taskManager.evaluateAll()` 与 `achievementManager.handleMetricsUpdated()`（桥接脚本已演示部分）。
4. 编写任务/成就面板，从 `taskManager.getAllTasks()` 和 `achievementManager.getAll()` 获取数据，
   根据状态显示不同按钮和标记。
5. 使用 `hasRewardableTask()` 和 `hasRewardable()` 驱动主入口红点和每日任务红点。
