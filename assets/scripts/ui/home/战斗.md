## 在大厅场景（HomeScene）建一个 UI 面板节点。

挂上这个 TeamFormationPanel 脚本。

在编辑器里拉两个 Layout 节点：一个作为左侧的仓库（Content），一个作为右侧的出战槽位。

随便用几张图片拼一个 SoulItem 的 Prefab（预制体），里面放上名字和 Cost 的 Text，把预制体拖给脚本。

只要在这个面板里把灵魂选好，点下【开始战斗】，你之前写的所有战斗底层代码就会瞬间激活，召唤师带着你选好的骷髅大军直接杀向战场！

## 方案 C：主程的“实机检验指南” (需你在 Cocos 里操作)
目标：代码写得再好，也要在画面上看到效果。不要急着把所有的面板都画完，我们要用最小成本验证整个战斗骨架！

当 Codex 帮你把前面的核心面板代码生成完之后，请你在 Cocos 编辑器里按以下步骤操作：

临时 UI 搭建：

打开 HomeScene。

新建一个 Canvas，随便放一个 Button 叫 “Start Battle”。

把 TeamFormationPanel.ts 挂在一个空节点上，把这个按钮拖给 btnStartBattle 属性。

（为了测试，你可以先不画左边的 ScrollView，直接在代码的 Start 里强行把 equippedSoulIds 写死成 2 个骷髅的 ID）。

场景跳转检查：

确保你的 Project -> Project Settings -> Scene 里面包含了 HomeScene 和 BattleScene。

检查 director.loadScene("BattleScene") 这行代码是否会报错。

战斗场景兜底：

确保 BattleScene 里面挂载的 BattleBootstrap 上的 autoStartBattle 已经打开。

确保 RemotePrefabCache 里面的预制体路径，和你配置表里填写的完全一致。
