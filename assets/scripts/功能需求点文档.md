火山女儿类养成游戏 功能需求点文档

## 0. 文档信息

- 文档名称：火山女儿类养成游戏 功能需求点文档  
- 版本：v1.0  
- 适用范围：单机养成+模拟经营+轻冒险的《火山女儿》风格游戏  
- 面向角色：策划、程序、UI、美术、测试

---

## 1. 游戏总体功能需求

### 1.1 游戏定位与目标

- 【F-001】游戏类型定义  
  - 支持配置“单机本地存档”模式，默认离线可玩。  
  - 支持多周目游玩，每个周目为一条独立的存档线。  
  - 核心玩法：女儿养成（日程安排、属性成长）+ 经济管理 + 社交 + 轻战斗/冒险。

- 【F-002】核心体验目标  
  - 玩家通过安排女儿的日常活动，影响其能力、性格、命运与结局。  
  - 玩家在有限时间内（例如 10～18 岁）平衡：属性成长 / 心情健康 / 经济收入 / 冒险风险。  
  - 同一女儿通过不同培养方式、不同事件选择，获得不同结局与人生路线。

- 【F-003】周目结构  
  - 支持设定单周目起始年龄与结束年龄（例如 10～18 岁）。  
  - 周目结束时根据数据判定结局，并显示结局统计与解锁状态。  
  - 周目结束后可开始新周目，部分信息（如已解锁结局图鉴）可在全局保留。

## 2. 核心循环与基础流程

### 2.1 每周循环结构

- 【F-010】周初概览界面  
  - 在每个“周”的开始展示概览界面，至少包含：  
    - 当前年份、季节、周次（如：第 X 年·春·第 Y 周）。  
    - 女儿年龄、关键主属性概要（如力量/智力/魅力等）。  
    - 当前金钱、体力、心情/疲劳度。  
    - 提示临近的关键事件（如节日、生日、主线节点）。  
  - 需支持从该界面进入日程安排、属性详情、家园等模块。

- 【F-011】每周日程安排入口  
  - 在周初必须允许玩家配置本周每日的行程。  
  - 支持进入“详细日程表”界面进行设置；未设置完毕不可直接跳到周末（需至少填满最基本要求的日程数量，具体可配置）。

- 【F-012】每日执行与结算流程  
  - 对于本周日程表中的每一天，按顺序执行：  
    1. 读取当天已安排的活动列表（若一天支持多时段则逐时段执行）。  
    2. 按活动顺序结算：属性变化、体力消耗、心情变化、金钱变化。  
    3. 检查是否触发与活动挂钩的事件（训练事件、打工事件、冒险事件等）。  
    4. 检查是否有基于日期的日常事件或隐藏事件需要插入。  
  - 若发生“强制事件”（节日、主线），需支持：  
    - 中断当前活动，先播放强制事件。  
    - 事件结束后继续当前活动或直接结束当日（依据事件配置）。

- 【F-013】周末事件流程  
  - 在一周结束时，根据以下条件触发周末事件：  
    - 周末特定时间条件（如每个月最后一周的竞赛）。  
    - 周内累计的数据（如工作时长、冒险次数）达到某阈值。  
    - 某些 NPC 的好感及 flag 达到触发条件。  
  - 支持多个事件排队播放，按优先级顺序执行：  
    - 高优先级：主线推进类事件。  
    - 中优先级：NPC 关键剧情事件。  
    - 低优先级：日常或随机事件。  
  - 事件执行完毕后，进入下一周周初界面并生成自动存档。

### 2.2 游戏主菜单与存档

- 【F-020】主菜单功能  
  - 至少包含：开始新游戏、读取存档、继续游戏（若有自动存档）、设置、结局图鉴、退出游戏。  
  - “开始新游戏”需支持：  
    - 输入女儿名字。  
    - 选择难度（如普通/困难）或在后续设置中调整。  
  - “读取存档”：显示存档列表，支持选中某一存档并进入对应进度。

- 【F-021】存档/读档功能  
  - 支持 N 个手动存档槽（N 可配置）。  
  - 提供自动存档槽，建议在每周开始时自动保存。  
  - 存档信息结构至少包含：  
    - 存档时间（现实时间）。  
    - 游戏内日期（年/季节/周）。  
    - 女儿年龄与若干关键属性数值。  
    - 当前主线进度标签或最近一次重大事件简述。  
  - 读档时需完整恢复：  
    - 女儿全部属性/状态。  
    - 事件触发状态（flag）。  
    - 当前金钱/物品。  
    - 当前周及日程安排（未执行部分）。

## 3. 女儿角色与属性系统

### 3.1 基础信息

- 【F-030】女儿基础信息显示  
  - UI 层面需要常驻区域显示：  
    - 姓名（可自定义）。  
    - 年龄（精确到岁，必要时显示年月）。  
    - 生日（用于生日事件判定）。  
    - 当前身份/称号（例如“火山之女”“王都学徒”）。  
  - 名字输入需：  
    - 限制最大长度（例如 8 个中文或 12 个英文字符）。  
    - 限制非法字符（不可包含控制字符等）。

- 【F-031】立绘与外观变化  
  - 为女儿提供至少按年龄段区分的立绘资源：  
    - 儿童期。  
    - 少年期。  
    - 成长期。  
  - 同时预留扩展：  
    - 根据职业/路线切换立绘（如骑士装、法师装、祭司装）。  
  - 系统需提供接口：  
    - 根据当前年龄与职业倾向，计算应该显示的立绘 ID。  
    - 支持临时覆盖（如在特定事件中穿礼服、祭服）。

### 3.2 属性结构

- 【F-032】基础属性定义与展示  
  - 至少定义以下基础属性（名称可根据具体设定微调）：  
    - 力量（STR）：影响物理攻击、体力劳动效率。  
    - 体质（VIT）：影响体力上限、抗病能力。  
    - 智力（INT）：影响学习效率、魔法威力。  
    - 魅力（CHA）：影响社交、演出、贵族应对等成功率。  
    - 艺术（ART）：影响绘画、音乐、舞蹈等艺术类表现。  
    - 信仰/魔力（FTH/MAG）：影响神秘、宗教、魔法相关事件走向。  
    - 勇气（CRG）：影响冒险相关事件中的特殊选项与成功率。  
  - 每项属性需：  
    - 设定最小值（通常为 0）。  
    - 设定初始值（根据开局设定决定）。  
    - 设定最大值（如 999，可在配置表中调整）。  
  - 属性展示界面：  
    - 提供简略视图（仅显示主属性当前值）。  
    - 提供详细视图（显示增长历史、加成来源拆解：基础值+装备+临时状态）。

- 【F-033】战斗属性与计算  
  - 定义战斗属性：HP、物理攻击、物理防御、魔法攻击、魔法防御、速度、命中、闪避等。  
  - 每个战斗属性由基础属性和装备属性通过可配置公式计算：  
    - 例：物理攻击 = 力量 × 系数 + 武器攻击力。  
    - 例：HP 上限 = 体质 × 系数 + 固定值。  
  - 需有统一接口进行战斗属性刷新，在以下时机调用：  
    - 基础属性变动后。  
    - 更换装备后。  
    - 获得/失去临时状态效果后。

- 【F-034】状态值  
  - 定义即时状态值：  
    - 体力：0–100，代表可行动程度。  
    - 心情：0–100，影响训练效率、事件选择倾向。  
    - 疲劳：0–100，影响生病几率、活动效果衰减。  
    - 健康状态：枚举（正常/轻病/重病/昏迷等）。  
  - 状态值变化规则：  
    - 体力随高强度活动快速下降，休息或使用道具恢复。  
    - 心情随成功事件提升，失败/训斥/加班等下降。  
    - 疲劳随连续活动累积，长时间休息才大幅降低。  
    - 健康状态由疲劳、事件、特殊 flag 决定。

### 3.3 属性变更与成长规则

- 【F-035】统一属性修改接口  
  - 程序侧需提供统一函数，如 `ModifyAttribute(id, delta, mode)`：  
    - `id`：属性标识。  
    - `delta`：变更值，可为正负。  
    - `mode`：变更模式（绝对加减、百分比变化、设定为某值）。  
  - 所有活动、事件、道具效果必须通过该接口进行属性变更，以便：  
    - 统一处理上限判断。  
    - 统一记录日志/回放数据（可选）。

- 【F-036】年龄成长系数  
  - 为每个基础属性配置“年龄成长系数表”：  
    - 例：力量在 10–14 岁成长系数 1.0，14–16 岁为 0.8，16–18 岁为 0.5。  
  - 当进行训练类属性增长时：  
    - 增加量 = 基础增长值 × 年龄系数 × 其他加成。  
  - 支持不同难度下采用不同的成长系数表。

- 【F-037】属性上限与锁定  
  - 每个属性包含一个可配置上限。  
  - 当调用属性修改接口时，若结果超过上限则：  
    - 实际值保持在上限。  
    - 可选：记录溢出量，用于特殊隐藏判定。  
  - 特定事件可设置属性锁定：  
    - 锁定后，属性不再改变或仅允许下降。  
    - 属性锁定状态需在 UI 中有可见标识。

- 【F-038】属性对系统的影响  
  - 属性用于多处系统判定：  
    - 事件触发条件（如魅力 ≥ 50 解锁舞会邀请）。  
    - 活动成功率（如智力越高学习效率越高）。  
    - 战斗表现（如勇气影响逃跑成功率）。  
  - 判定条件应数据化配置，避免硬编码。

## 4. 时间与日历系统

### 4.1 时间结构

- 【F-040】日期结构与推进  
  - 定义统一时间结构：年、季节、周、日。  
  - 内部计时可采用“绝对日数”以简化计算，并映射到上述结构。  
  - 每执行完一日活动后，自动将当前日 +1，并刷新对应季节与周。  
  - 当年数增加时，自动更新女儿年龄。

- 【F-041】年龄与关键节点  
  - 为年龄定义关键节点（例如 12、14、16、18 岁）。  
  - 到达关键节点时：  
    - 自动触发检查函数，判定是否有对应主线或职业分支事件可执行。  
    - 记录“已通过关键节点”的 flag，用于结局判定。

### 4.2 日历与节日

- 【F-042】节日数据结构  
  - 每个节日配置：  
    - 日期（年内第几天或季节+日）。  
    - 名称、类型（宗教/庆典/比赛/纪念日）。  
    - 是否强制事件（强制参加或强制限制日程）。  
    - 关联系统（如比赛入口 ID、宗教仪式事件 ID）。

- 【F-043】日历查看功能  
  - 提供一个全年日历 UI：  
    - 高亮显示节日与重要事件。  
    - 鼠标悬停（或选择）时显示节日简介。  
  - 支持标记玩家已知的预约事件日期（如某 NPC 邀约的约会日）。

## 5. 日程与活动系统

### 5.1 日程表功能

- 【F-050】每周日程表结构  
  - 支持配置每周的日程槽数量：  
    - 简化版：每周 7 天，每天 1 个活动槽。  
    - 扩展版：每天 2–3 个时间段（上午/下午/晚上）。  
  - 日程表需要：  
    - 可视化展示每一天的安排。  
    - 支持查看每个活动的简要信息（属性增减、体力消耗等）。

- 【F-051】日程编辑交互  
  - 活动列表区域：按类别展示可选活动（训练/打工/冒险/休息/特殊）。  
  - 日程格操作：  
    - 左键点击格子弹出活动选择列表。  
    - 或将活动从列表拖拽到格子。  
  - 支持快捷操作：  
    - “复制上周日程”按钮，将上一周的安排整体复制。  
    - “清空本周日程”按钮，将当前周所有格子置空。

- 【F-052】日程校验与提示  
  - 在玩家点击“确认本周日程”时进行校验：  
    - 检查是否和节日/强制事件冲突。  
    - 检查是否安排了当前不可用活动（未解锁或属性不足）。  
  - 对每个问题给出明确提示：  
    - 冲突活动会被标红并提示原因。  
    - 提供“自动修正”选项（如将冲突项改为休息）。

### 5.2 活动定义与结算

- 【F-053】活动配置字段  
  - 活动表字段参考：  
    - 活动ID、名称、类型。  
    - 前置条件（日期区间、属性阈值、好感阈值、金钱、flag）。  
    - 基础消耗（体力、疲劳、金钱）。  
    - 基础收益（属性增减、金钱、好感、特殊点数）。  
    - 成功率相关参数（基础成功率、不同等级成功的几率）。  
    - 关联事件 ID 列表（触发概率、是否唯一）。

- 【F-054】活动执行顺序与逻辑  
  - 当执行某项活动时：  
    1. 校验前置条件：若不满足则活动失败，进入失败分支或替换为休息。  
    2. 根据配置扣除消耗（体力/金钱），若扣除后体力为负，触发超负荷事件。  
    3. 按成功率计算结果等级（失败/普通成功/大成功）。  
    4. 根据等级应用不同的属性/金钱/好感变化。  
    5. 判断是否触发关联事件（按配置概率与条件）。

- 【F-055】活动解锁与进阶  
  - 活动被解锁的方式：  
    - 女儿属性达到指定阈值。  
    - 时间推进到特定年/季节。  
    - 完成相关主线或支线事件。  
    - 与某 NPC 好感达到一定等级。  
  - 解锁后：  
    - 在活动列表中显示，并标注“NEW”。  
    - 相关 NPC 或系统可通过事件提示玩家。

## 6. 经济与物品系统

### 6.1 货币与收支

- 【F-060】货币系统  
  - 定义主货币（金币）：用于课程、道具、装备、家园升级等。  
  - 如需扩展，可添加：  
    - 奉献点（用于宗教/神殿系统）。  
    - 声望值（用于高阶社交、贵族圈层）。

- 【F-061】收入来源明细  
  - 打工类活动：  
    - 每次完成后固定或区间随机获得金币。  
    - 收入与相关属性（如魅力/力量）挂钩。  
  - 冒险收益：  
    - 掉落装备与素材，可回城后在商店出售。  
  - 事件奖励：  
    - 比赛获胜、任务完成、贵族赏赐等一次性收入。

- 【F-062】支出场景细化  
  - 教育支出：报名费、学费、教材费。  
  - 装备与道具支出：购买新装备、药剂、礼物。  
  - 生活支出（可选）：日常伙食、房租/税金等，用于增加经济压力与真实感。

### 6.2 背包与物品

- 【F-063】物品分类与标签  
  - 装备：武器、防具、饰品，包含攻击、防御、附加属性等字段。  
  - 消耗品：药剂、食物、临时 buff 道具。  
  - 礼物：带有 NPC 喜好标签（喜欢/普通/讨厌）。  
  - 任务物品：与特定事件绑定，不能随意丢弃/出售。

- 【F-064】背包界面与交互  
  - 提供类别筛选按钮（全部/装备/消耗品/礼物/任务）。  
  - 提供排序方式（品质、名称、数量、获取时间）。  
  - 单击物品展示简略信息，双击或按“使用”按钮执行使用逻辑。  
  - 对不可使用的物品（当前场景不允许）需灰显并提示原因。

- 【F-065】物品使用与装备操作  
  - 消耗品使用：立即生效，调用统一状态/属性修改接口。  
  - 装备操作：  
    - 打开装备界面，显示当前装备栏位（武器/防具/饰品等）。  
    - 选择背包中的装备替换当前装备，实时刷新战斗属性。  
  - 礼物使用：通过“送礼”界面选择目标 NPC，再调用好感变化逻辑。

- 【F-066】商店系统细化  
  - 商店列表数据：商品 ID、价格、库存、是否限购、上架条件。  
  - 价格可根据难度/声望/事件变化（如打折活动）动态调整。  
  - 支持出售功能：玩家可将背包物品出售获得金币（部分物品不可卖）。

## 7. 冒险与战斗系统

### 7.1 冒险入口与准备

- 【F-070】冒险选择界面  
  - 从日程或家园进入“冒险”模块。  
  - 玩家选择：  
    - 目标区域（森林/山脉/火山等）。  
    - 难度等级（初级/中级/高级）。  
  - 界面显示：  
    - 区域推荐等级。  
    - 可能的掉落类型。  
    - 预计风险提示。

- 【F-071】出发前检查  
  - 检查女儿当前体力是否高于某阈值，不足则弹出风险提醒。  
  - 检查女儿装备情况（如未装备武器时给出提示）。  
  - 玩家确认后才能开始冒险。

### 7.2 探索与战斗

- 【F-072】简化探索流程  
  - 使用节点式地图：每次前进至下一个节点触发一个事件（怪物战/宝箱/剧情）。  
  - 允许玩家在节点间选择“前进”或“返回”以控制风险。  
  - 根据当前区域配置抽取事件：  
    - 普通战斗事件。  
    - 精英战/首领战。  
    - 宝箱或资源点。  
    - 陷阱或负面事件。  
    - 特殊剧情节点。

- 【F-073】战斗系统核心  
  - 采用回合制：  
    - 回合开始时按速度属性决定行动顺序。  
    - 玩家在自己的回合可选择：攻击、技能、道具、防御、逃跑。  
  - 技能需有：消耗（体力/魔力）、效果（伤害/治疗/增益/减益）、命中率、冷却等字段。  
  - 伤害计算：  
    - 使用攻防属性和技能系数计算。  
    - 考虑暴击、属性克制等可选要素。

- 【F-074】战斗结算  
  - 胜利：  
    - 获得经验（可转化为属性成长或战斗熟练度）。  
    - 掉落金币和物品。  
  - 失败：  
    - 女儿被送回家，扣除部分金币。  
    - 增加疲劳，可能附加“受伤”状态。  
  - 结算后根据玩家选择决定是继续深入还是返回。

## 8. 社交与好感系统

### 8.1 NPC 结构与信息

- 【F-080】NPC 分类与配置  
  - 为每个 NPC 配置：  
    - 基本信息：姓名、年龄（选）、身份、职业。  
    - 立绘资源 ID。  
    - 初始好感度与好感上限。  
    - 喜好标签（喜欢/讨厌的礼物类型、活动类型）。  
    - 关联事件 ID 列表（好感阶段解锁事件）。

- 【F-081】NPC 信息界面  
  - 提供 NPC 名册界面：  
    - 展示已认识 NPC 的头像和名字。  
    - 未认识 NPC 显示为“？？？”。  
  - 选中 NPC 显示详细信息：  
    - 当前好感数值和等级标记。  
    - 已解锁的背景信息、关键事件纪要。  
    - 若有对应结局线，可在不剧透的前提下给出模糊提示。

### 8.2 好感度与互动

- 【F-082】好感度数值与等级  
  - 每个 NPC 拥有 0–100（或其他区间）的好感值。  
  - 按区间划分等级：陌生/认识/朋友/挚友/恋人等。  
  - 界面需以颜色或图标体现等级变化。

- 【F-083】好感变化来源  
  - 日常对话选项：不同回答修改好感。  
  - 共同参与活动：如节日、特定训练/打工场景。  
  - 送礼：根据礼物与 NPC 喜好匹配度调整好感变化幅度。  
  - 剧情事件：关键选择支持一次性大幅修改好感或锁死关系状态。

- 【F-084】社交事件与约会系统（可选）  
  - 特定好感等级后解锁约会/私下见面事件。  
  - 约会中根据选项进一步影响好感与未来事件。

## 9. 事件、剧情与多结局

### 9.1 事件系统

- 【F-090】事件数据结构  
  - 每个事件包含：  
    - 事件ID、名称、类型（主线/支线/日常/特殊）。  
    - 触发条件（时间、地点、属性、好感、flag）。  
    - 是否唯一事件（仅触发一次）、是否可重复。  
    - 事件脚本引用（对话内容、选项、结果）。

- 【F-091】事件触发与队列调度  
  - 在每日/每周结算关键节点调用事件触发器：  
    - 扫描所有满足条件的事件。  
    - 按优先级和权重构建事件队列。  
  - 一次只播放一个事件，播放完后再检查是否继续播放队列中的下一个。

### 9.2 剧情分支与选项

- 【F-092】分支选项系统  
  - 每个事件可以包含多个选项：  
    - 每个选项配置前置条件（属性/好感/flag）。  
    - 不满足条件的选项应灰显或隐藏。  
  - 选项结果：  
    - 修改属性、好感、金钱。  
    - 设置/清除事件 flag。  
    - 跳转到其他事件节点或结束事件。

### 9.3 多结局

- 【F-093】结局条件与优先级  
  - 每个结局配置：  
    - 结局ID、名称、类型（职业/感情/命运等）。  
    - 判定条件集合（属性、好感、关键事件 flag）。  
    - 优先级权重。

- 【F-094】结局判定流程  
  - 在女儿达到最终年龄（如 18 岁）时：  
    1. 收集所有满足条件的结局列表。  
    2. 按优先级从高到低选择第一个满足的结局。  
    3. 记录该结局为已解锁，并展示对应 CG 与文本。

- 【F-095】结局图鉴  
  - 在主菜单提供结局图鉴界面：  
    - 显示已解锁/未解锁结局数量。  
    - 已解锁结局可回看结局描述与 CG。  
    - 未解锁结局显示为锁定状态，可仅显示模糊提示（如“某职业相关结局”）。

## 10. 家园与养成互动

### 10.1 家园主界面

- 【F-100】家园场景结构  
  - 家园作为主要非战斗/非日程场景，包含：客厅、女儿房间、庭院等点击区域。  
  - 在家园可访问：属性界面、背包、装备、对话、家园升级等功能。

### 10.2 与女儿的互动

- 【F-101】日常对话系统  
  - 在家时点击女儿立绘可触发对话：  
    - 对话文本根据当前心情、疲劳、事件进度切换。  
    - 支持随机台词池与状态敏感台词。  
  - 对话中可插入简单选项影响心情或轻微属性。

- 【F-102】教育态度选择  
  - 在特定对话节点给予玩家“鼓励/表扬/批评/严厉”等选项。  
  - 这些选项影响：  
    - 女儿短期心情。  
    - 长期性格倾向（如顺从/叛逆、外向/内向）。  
  - 性格倾向进一步影响未来事件中的选项偏好与表现。

### 10.3 家园升级

- 【F-103】家具与装饰系统  
  - 玩家可消费金币购买家具与装饰物。  
  - 家具安装位置预设插槽，例如：墙面、地面、桌面等。  
  - 不同家具提供：  
    - 被动属性加成（如学习效率+X）。  
    - 解锁特殊互动或事件（如钢琴开启音乐训练事件）。

- 【F-104】家园外观变化  
  - 根据家园升级进度，切换不同背景资源：简陋房屋→普通住宅→宽敞宅邸。  
  - 外观变化对事件和 NPC 评价产生影响（如贵族更愿意来访）。

## 11. 系统设置与辅助功能

### 11.1 设置菜单

- 【F-110】音频设置  
  - 可分别调节 BGM、SE、语音音量。  
  - 提供一键静音选项。

- 【F-111】显示与性能设置  
  - 窗口化/全屏切换。  
  - 分辨率选择（列出常见分辨率）。

  - 文本自动播放速度调节。  
  - 是否跳过已读文本。  
  - 是否启用战斗加速/动画跳过。

### 11.2 帮助与引导

- 【F-113】新手引导  
  - 在前几个周目关键节点弹出说明：  
    - 属性的基础含义。  
    - 日程安排和活动分类。  
    - 简单经济与社交系统说明。  
  - 支持玩家在设置中关闭进一步引导。

- 【F-114】帮助说明界面  
  - 提供一个集中帮助页面，按模块解释：  
    - 属性/状态的功能。  
    - 各类活动的大致作用。  
    - 冒险/战斗的基础逻辑。

  - 活动、事件、NPC、物品、结局、节日等全部从配置表加载。  
  - 游戏逻辑通过读取配置表控制行为而非硬编码数值。  
  - 配置表支持热更新或方便策划快速迭代。

- 【F-116】多语言与本地化  
  - 所有文本（UI、对话、帮助）从本地化资源表读取。  
  - UI 布局需预留不同语言字数差异空间。

## 12. 示例：一周流程拆解

- 【F-120】一周流程示例  
  1. 周初：弹出周概览 → 玩家查看状态。  
  2. 玩家进入日程界面 → 填写本周 7 天活动 → 系统校验并确认。  
  3. 执行每天活动：按顺序结算活动效果，并插入必要事件。  
  4. 周末：检查节日/主线/社交事件 → 播放事件队列。  
  5. 自动存档 → 进入下一周周初。
