# 火山女儿(Volcano Girl) 游戏战斗系统分析

## 一、游戏概述
火山女儿是一款经典的角色扮演卡牌策略游戏，融合了RPG和卡牌两大元素，具有丰富的战斗系统。

---

## 二、战斗系统核心需求

### 1. 战斗的基本模式

#### 1.1 战斗场景类型
- **副本关卡战斗** - PvE模式，对战AI敌人
- **竞技场战斗** - PvP模式，对战其他玩家阵容
- **BOSS战** - 特殊配置的高难度敌人
- **活动副本** - 限时或特殊规则的战斗

#### 1.2 战斗阶段
1. **出征前准备阶段**
   - 选择出战英雄（队伍配置）
   - 查看敌方阵容
   - 选择辅助技能/道具
   
2. **战斗开始阶段**
   - 英雄上阵动画
   - 敌方登场
   
3. **战斗进行阶段** (核心)
   - 回合制战斗
   - 技能选择与释放
   - 伤害计算与结果反馈
   
4. **战斗结束阶段**
   - 胜负判定
   - 掉落物品统计
   - 经验值获得
   - 战斗回放

## 三、战斗系统详细设计

### 1. 队伍与阵容系统

#### 1.1 队伍结构
```
玩家队伍 (最多5个英雄)
├── 前排位置 (1-3号位，肉盾/坦克)
├── 中排位置 (2-4号位，均衡)
└── 后排位置 (3-5号位，输出/治疗/辅助)

敌方队伍 (同结构)
```

#### 1.2 位置影响
- **前排**：受伤害减少，但输出较低
- **中排**：均衡属性
- **后排**：输出高但脆弱，容易被击中

### 2. 战斗流程

#### 2.1 回合制流程
```
┌─ 一个回合的流程 ──────────────────────┐
│                                        │
│ 1. 速度排序 (Speed >= 0的英雄)        │
│    ├─ 按英雄速度值从高到低排序       │
│    └─ 同速度则按阵容顺序位置排序     │
│                                        │
│ 2. 依次出手                             │
│    ├─ 选择行动 (普通攻击/技能/防守)   │
│    ├─ 执行行动                        │
│    ├─ 伤害计算与应用                  │
│    ├─ 状态效果判定 (眩晕/燃烧/etc)  │
│    └─ 动画播放与反馈                  │
│                                        │
│ 3. 检查战斗结束条件                    │
│    ├─ 某一方全灭 → 胜负判定           │
│    ├─ 达到最大回合数 → 判定结果       │
│    └─ 其他特殊条件                    │
│                                        │
│ 4. 继续下一回合或结束                  │
└────────────────────────────────────────┘
```

#### 2.2 英雄行动选择
每个英雄的回合可选行动：
1. **普通攻击** (Normal Attack)
   - 消耗：无
   - 伤害：基础伤害值
   - 特效：可能触发连击/暴击

2. **技能释放** (Skill)
   - 消耗：法力值(MP)或技能点数(SP)
   - 伤害/治疗：技能倍率计算
   - 特效：多种效果(群体/单体/持续伤害等)

3. **防守** (Defend)
   - 消耗：无
   - 效果：防御力提升，减少受伤

4. **逃跑** (Escape) - 仅限特定游戏模式
   - 消耗：无
   - 效果：尝试逃离战斗

### 3. 伤害计算系统

#### 3.1 伤害公式
```
基础伤害 = (攻击者攻击力 - 防御者防御力) * 技能倍率 * 属性相克倍率
           + 额外加成(装备/BUFF/技能加成)

最终伤害 = 基础伤害 * (1 + 暴击伤害倍率) * (1 + 连击倍率)
         * 位置克制倍率 * 特殊效果倍率

受伤害 = MAX(1, 最终伤害) * (1 - 防御减伤%)
```

#### 3.2 暴击系统
- **暴击率** (Crit Rate): 触发暴击的概率
- **暴击倍率** (Crit Damage): 暴击时伤害倍数(如1.5x)
- 暴击判定：随机数 < 暴击率 → 触发

#### 3.3 连击系统
- **连击率** (Combo Rate): 触发连击的概率
- **连击倍率** (Combo Damage): 连击数量和倍数
- 可连续触发多次连击

### 4. 属性与克制系统

#### 4.1 属性类型 (6种或5种)
- 火 (Fire) - 攻击力高
- 水 (Water) - 防御力高
- 木 (Wood) - 速度快
- 光 (Light) - 综合
- 暗 (Dark) - 综合
- (可选额外属性)

#### 4.2 属性克制关系
```
火 > 木 > 水 > 火  (形成循环)
光 > 暗，暗 > 光  (对立)
```

#### 4.3 克制倍率
- 克制关系: 伤害 × 1.2-1.5 (有利)
- 反克关系: 伤害 × 0.8-0.5 (不利)
- 无关系: 伤害 × 1.0 (正常)

### 5. 状态效果系统 (Buff/Debuff)

#### 5.1 常见负面状态 (Debuff)
| 状态 | 效果 | 持续时间 | 可叠加 |
|------|------|---------|-------|
| 眩晕 (Stun) | 跳过该回合 | 1-3回合 | 否 |
| 燃烧 (Burn) | 每回合损血 | 2-4回合 | 是 |
| 冰冻 (Freeze) | 速度降低 | 1-2回合 | 否 |
| 中毒 (Poison) | 每回合掉血 | 持续 | 是 |
| 虚弱 (Weak) | 攻击力降低 | 2-3回合 | 是 |
| 失血 (Bleed) | 持续掉血 | 3-5回合 | 是 |

#### 5.2 常见正面状态 (Buff)
| 状态 | 效果 | 持续时间 |
|------|------|---------|
| 强化 (Enhance) | 攻击力提升 | 2-3回合 |
| 防御 (Defense) | 防御力提升 | 2-3回合 |
| 加速 (Haste) | 速度提升 | 2-3回合 |
| 回复 (Regen) | 每回合回血 | 3-5回合 |
| 护盾 (Shield) | 抵消伤害 | 1回合或固定值 |
| 无敌 (Invincible) | 免疫伤害 | 1回合 |

#### 5.3 状态管理
- 状态叠加规则
- 状态覆盖规则
- 状态优先级

## 四、英雄系统

### 1. 英雄属性
```typescript
interface Hero {
    // 基础信息
    id: number;              // 英雄ID
    name: string;            // 英雄名称
    rarity: number;          // 稀有度 (1-5星)
    element: string;         // 属性 (火/水/木/光/暗)
    
    // 战斗属性
    hp: number;              // 生命值
    attack: number;          // 攻击力
    defense: number;         // 防御力
    speed: number;           // 速度
    
    // 特殊属性
    critRate: number;        // 暴击率 (0-100%)
    critDamage: number;      // 暴击伤害倍率
    hitRate: number;         // 命中率
    dodgeRate: number;       // 闪避率
    
    // 技能
    skills: Skill[];         // 技能列表
    passiveSkill: Skill;     // 被动技能
    
    // 装备
    equipment: Equipment[];  // 装备列表
}
```

### 2. 英雄等级与进阶
- **等级系统** (Level 1-100)
  - 等级提升属性
  - 获得技能点数
  
- **进阶/突破系统** (Star + 阶位)
  - 消耗材料进阶
  - 属性大幅提升
  - 解锁新技能或加强

### 3. 技能系统

#### 3.1 技能属性
```typescript
interface Skill {
    id: number;
    name: string;
    description: string;
    
    // 消耗
    costType: "MP" | "SP";  // 消耗类型
    costValue: number;      // 消耗数值
    
    // 效果
    type: "damage" | "heal" | "buff" | "debuff" | "mixed";
    target: "single" | "all" | "random" | "self";
    damageRatio: number;    // 伤害倍率 (100 = 100%)
    
    // 特殊效果
    effects: StatusEffect[]; // 附加的状态效果
    
    // 其他
    cooldown: number;       // 冷却回合数
    priority: number;       // 技能优先级
}
```

#### 3.2 技能分类
- **物理技能** - 基于攻击力
- **魔法技能** - 基于特殊魔法属性
- **治疗技能** - 回复血量
- **辅助技能** - 增益效果
- **控制技能** - 眩晕/减速等

## 六、游戏平衡性需求

### 1. 难度设置
- **简单 (Easy)**：敌人属性降低30%
- **普通 (Normal)**：标准设置
- **困难 (Hard)**：敌人属性提升50%
- **地狱 (Hell)**：敌人属性提升100%+

### 2. 胜率平衡
- 等级压制规则：等级差影响伤害倍率
- 属性克制平衡：克制倍率调整
- 技能冷却平衡：防止某些技能过于强势

### 3. 掉落与奖励
- **副本掉落**
  - 普通掉落（金币、经验）
  - 稀有掉落（英雄碎片、装备）
  - 必定掉落（任务物品）

- **奖励机制**
  - 首次通关奖励 (首次通关获得额外奖励)
  - 星级奖励 (3星通关额外奖励)
  - 连续通关奖励

7. **是否需要实时PvP？** → 建议先做PvE
8. **是否需要数据库存储战斗日志？** → 建议支持
