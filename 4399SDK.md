# 4399SDK 高级小游戏 SDK 框架设计方案

## 1. 文档目标

### 1.1 框架边界

`H` 框架内的 SDK 能力必须保持项目无关，只做通用能力封装：

- `H.sdk` 只负责登录、分享、收藏、复访、侧边栏、添加桌面等平台能力，并返回统一结果。
- `H.ad` 只负责广告 ID 读取、广告预加载、广告展示、冷却、防重复调用、失败归因和统计。
- `H.reward` 只负责“是否满足发奖条件”的通用校验、次数、冷却、幂等记录，不写任何具体奖励内容。
- 框架不得写复活、金币、体力、关卡、皮肤、任务进度等项目业务逻辑。
- 具体项目通过配置传入广告位、分享文案、功能开关和奖励 key；通过业务层根据 `rewarded` / `rewardable` / `granted` 自己发放实际奖励。

文档中的 `revive`、`double_coin`、`level_end` 等名称只作为接入示例，不允许作为框架默认配置或框架代码常量。

本文档用于指导当前项目的 `H` 框架升级为一套可复用、可运营、可商业化的高级小游戏 SDK。

目标不是简单封装 `wx`、`tt`、`4399` 的平台 API，而是形成一套业务层可以直接使用的统一框架：

```ts
H.sdk          // 平台能力：登录、分享、收藏、复访、侧边栏、添加桌面、录屏
H.ad           // 广告能力：激励视频、插屏、Banner、原生广告
H.reward       // 统一发奖：广告奖励、分享奖励、收藏奖励、任务奖励
H.config       // 渠道配置：广告 ID、分享文案、功能开关、冷却时间
H.analytics    // 埋点上报：广告、登录、分享、留存、时长
H.account      // 账号态：游客、平台登录、服务器 token
H.session      // 会话时间：进入时间、前后台、在线时长、冷却锁
H.compliance   // 合规能力：隐私、实名认证、防沉迷预留
```

业务层原则：

- 不直接调用 `wx`、`tt`、`window.xxx`。
- 不直接保存平台登录返回。
- 不直接写广告 ID。
- 不直接根据平台写分支逻辑。
- 不在业务代码里散写发奖、防重复、冷却和埋点逻辑。

## 2. 当前项目现状

当前项目已经具备以下基础：

- `assets/scripts/H/H.ts` 已有统一入口。
- `H.platform` 已能识别微信、抖音、Mock。
- `H.ad` 已封装激励视频、插屏、Banner。
- `H.data` 已有模块化本地存储和切后台落盘。
- `H.user` 已存储头像、昵称、定位等用户信息。
- `H.session` 已新增真实会话时间、冷却、循环任务能力。
- `H.sdk` 已新增登录、分享、收藏、复访、侧边栏、添加桌面等统一入口。
- Loading 已接入开局自动登录，并把结果保存到 `platformSession`。

当前缺口：

- 没有统一配置中心。
- 没有平台能力检测表。
- 没有统一发奖系统。
- 没有服务端发奖校验和幂等。
- 没有埋点上报系统。
- 没有远程配置和线上开关。
- 没有账号 token、游客账号、刷新登录态。
- 没有隐私、实名、防沉迷等合规层。
- 没有抖音录屏分享、排行榜、好友邀请等增长能力。

## 3. 总体架构

推荐架构如下：

```text
业务层
  ↓
H.reward / H.sdk / H.ad / H.account
  ↓
H.config / H.session / H.analytics / H.data
  ↓
平台 Adapter：Wechat / Douyin / 4399 / Web / Mock
  ↓
真实平台 API：wx / tt / h5 / 4399 / browser
```

职责划分：

| 模块 | 职责 |
| --- | --- |
| `H.sdk` | 平台能力统一入口 |
| `H.ad` | 广告能力统一入口 |
| `H.reward` | 统一发奖、幂等、次数、冷却 |
| `H.config` | 本地配置、远程配置、功能开关 |
| `H.analytics` | 行为埋点、错误上报、广告漏斗 |
| `H.account` | 平台登录、服务器登录、token 管理 |
| `H.session` | 会话时间、前后台、在线时长、冷却锁 |
| `H.compliance` | 隐私协议、实名、防沉迷预留 |
| `Adapter` | 微信、抖音、4399、Web、Mock 平台差异 |

## 5. 平台适配层

所有平台都实现同一套接口。

```ts
export interface HMiniGameSDKAdapter {
    readonly platform: 'wechat' | 'douyin' | '4399' | 'web' | 'mock';

    init(config: HSDKInitOptions): void;
    hasFeature(feature: HSDKFeature): boolean;

    login(): Promise<HSDKLoginResult>;

    share(options: HShareOptions): Promise<HSDKActionResult>;
    showFavoriteGuide(options?: HFavoriteGuideOptions): Promise<HSDKActionResult>;
    showRevisitGuide(options?: HRevisitGuideOptions): Promise<HSDKActionResult>;
    checkSidebar(options?: HSidebarOptions): Promise<HSDKActionResult>;
    navigateToSidebar(options?: HSidebarOptions): Promise<HSDKActionResult>;
    checkShortcut(options?: HShortcutOptions): Promise<HSDKActionResult>;
    addShortcut(options?: HShortcutOptions): Promise<HSDKActionResult>;

    startRecord?(options?: HRecordOptions): Promise<HSDKActionResult>;
    stopRecord?(): Promise<HRecordResult>;
    shareVideo?(options?: HShareVideoOptions): Promise<HSDKActionResult>;
}
```

平台差异必须只存在 Adapter 内部。

业务层只调用：

```ts
await H.sdk.share();
await H.sdk.addShortcut();
await H.sdk.showFavoriteGuide();
```

SDK 内部实现关系：

```text
H.sdk / HSDKFacade
  -> HDefaultSDKRegistry / HSDKRegistry
  -> HMiniGameSDKAdapter
    -> HWxSDK  -> wx.*
    -> HDySDK  -> tt.*
    -> HMockSDK
```

新增平台不修改 `HSDKFacade`，只注册平台 SDK：

```ts
H.sdk.registerAdapterClass(H4399SDK);
```

平台标识写在平台 SDK 类内部：

```ts
export class H4399SDK extends HBaseSDK {
    public readonly platform = '4399' as const;
}
```

## 6. 能力检测系统

### 6.1 屏幕适配系统

框架提供 `H.screen` 和 `H.sdk` 转发入口，用于统一获取微信、抖音、Mock/Web 环境下的屏幕尺寸与安全区：

```ts
const info = await H.sdk.refreshScreenInfo();

console.log(info.screenWidth, info.screenHeight);   // 设备实际屏幕尺寸
console.log(info.windowWidth, info.windowHeight);   // 小游戏可用窗口尺寸
console.log(info.safeAreaInsetsVisible);            // 转换到 Cocos 可视尺寸后的安全区
```

Loading 初始化时可配置设计尺寸：

```ts
H.init({
    screen: {
        designWidth: 720,
        designHeight: 1280,
        fitMode: 'auto',
        autoRefresh: true,
    },
});
```

业务 UI 需要适配时读取统一结果：

```ts
const adapt = H.sdk.getScreenAdaptResult({
    designWidth: 720,
    designHeight: 1280,
    fitMode: 'auto',
});

node.setScale(adapt.uiScale, adapt.uiScale, 1);
```

如果需要由框架设置 Cocos 设计分辨率：

```ts
H.sdk.applyScreenDesignResolution({
    designWidth: 720,
    designHeight: 1280,
    fitMode: 'auto',
});
```

业务层不得直接调用 `wx.getSystemInfoSync()`、`tt.getSystemInfoSync()` 来写平台分支。

不同平台支持的能力不一致，业务层必须先通过统一接口判断。

```ts
if (H.sdk.hasFeature('shortcut')) {
    this.showAddDesktopButton();
}

if (H.sdk.hasFeature('favorite')) {
    this.showFavoriteRewardButton();
}
```

## 8. 远程配置

远程配置用于线上开关、灰度、修复渠道异常。

典型使用：

```ts
if (!H.config.isFeatureEnabled('interstitial')) {
    return;
}
```

远程配置建议字段：

```ts
interface HRemoteConfig {
    version: number;
    updatedAt: number;
    switches: Record<string, boolean>;
    cooldown: Record<string, number>;
    ads?: Partial<HAdIds>;
    share?: Record<string, HShareOptions>;
}
```

加载流程：

```text
1. Loading 阶段读取本地缓存配置。
2. 使用缓存配置先进入游戏。
3. 后台请求远程配置。
4. 校验版本和结构。
5. 更新 H.config。
6. 写入 H.data。
```

这样即使远程配置请求失败，也不会阻塞首屏。

## 9. 统一结果结构

平台能力统一返回：

```ts
export interface HSDKActionResult {
    ok: boolean;           // API 调用是否成功
    completed: boolean;    // 玩家是否完成目标动作
    rewardable: boolean;   // 是否允许进入发奖流程
    platform: HResolvedPlatform;
    action: HSDKActionType;
    reason?: HSDKActionReason;
    userMessage?: string;
    raw?: unknown;
}
```

广告激励结果：

```ts
export interface HAdRewardResult {
    ok: boolean;
    shown: boolean;
    rewarded: boolean;     // 激励广告发奖只看这个
    completed: boolean;
    placement: string;
    platform: HResolvedPlatform;
    reason?: HAdFailReason;
    userMessage?: string;
}
```

发奖判断规则：

```ts
// 激励广告
if (adRet.rewarded) {
    // 可以发奖
}

// 分享、收藏、添加桌面、侧边栏
if (sdkRet.rewardable) {
    // 可以发奖
}
```

## 10. 统一发奖系统

业务不应在各处直接发奖。推荐统一走 `H.reward`。

广告发奖：

```ts
const ret = await H.reward.claimByRewardAd({
    rewardScene: 'revive',
    rewardId: 'revive_once',
    placement: 'revive',
});

if (ret.granted) {
    this.revivePlayer();
}
```

分享发奖：

```ts
const ret = await H.reward.claimBySDKAction({
    rewardScene: 'share_daily',
    rewardId: 'daily_share_1',
    action: () => H.sdk.share({
        title: '快来一起玩',
        rewardPolicy: 'completed',
    }),
});

if (ret.granted) {
    this.addDiamond(20);
}
```

发奖结果：

```ts
export interface HRewardResult {
    ok: boolean;
    granted: boolean;
    duplicated: boolean;
    rewardScene: string;
    rewardId: string;
    reason?: HRewardReason;
    userMessage?: string;
    raw?: unknown;
}
```

发奖原因：

```ts
export type HRewardReason =
    | 'sdk-not-completed'
    | 'ad-not-rewarded'
    | 'cooldown'
    | 'daily-limit'
    | 'duplicated'
    | 'server-rejected'
    | 'network-error'
    | 'config-disabled';
```

## 11. 发奖安全流程

客户端不能完全可信，尤其分享、收藏、添加桌面这类能力。

本地版流程：

```text
1. 业务请求发奖。
2. H.reward 检查功能开关。
3. H.reward 检查冷却。
4. H.reward 检查每日次数。
5. H.reward 调用 H.sdk 或 H.ad。
6. 根据 rewarded 或 rewardable 判断。
7. 写入本地领取记录。
8. 返回 granted=true。
```

服务端版流程：

```text
1. 业务请求发奖。
2. H.reward 创建 rewardToken。
3. 调用 H.sdk 或 H.ad。
4. SDK 返回完成结果。
5. H.reward 提交 rewardToken、uid、rewardScene、rewardId 到服务器。
6. 服务器检查账号、次数、幂等、风控。
7. 服务器返回 granted=true。
8. 客户端发放奖励或刷新背包。
```

服务端请求示例：

```ts
POST /reward/claim

{
    uid: '10001',
    token: 'server-token',
    platform: 'douyin',
    rewardScene: 'daily_share',
    rewardId: 'daily_share_1',
    rewardToken: 'uuid',
    action: 'share',
    clientTime: 123456789
}
```

## 12. 登录与账号系统

当前 Loading 已经可以调用：

```ts
await H.sdk.login();
```

后续建议升级为：

```ts
const account = await H.account.login();
```

内部流程：

```text
1. H.sdk.login 获取平台 code。
2. H.account 把 code 发到服务器。
3. 服务器返回 uid、token、openid。
4. SDK 保存 token 和过期时间。
5. token 过期自动 refresh。
6. 登录失败时进入游客模式。
```

账号数据结构：

```ts
export interface HAccountData {
    uid: string;
    platform: HResolvedPlatform;
    openid?: string;
    unionid?: string;
    token?: string;
    expiresAt?: number;
    isGuest: boolean;
    loginAt: number;
}
```

没有服务器时，先存平台返回：

```ts
{
    platform: 'douyin',
    login: {
        ok: true,
        code: 'xxx',
        anonymousCode: 'xxx'
    },
    updatedAt: Date.now()
}
```

## 14. 分享、收藏、添加桌面、侧边栏奖励

非广告行为统一看 `rewardable`。

```ts
const ret = await H.sdk.addShortcut({
    rewardPolicy: 'completed',
});

if (ret.rewardable) {
    this.addDiamond(50);
}
```

注意事项：

- 分享通常只能确认“调起成功/回调成功”，不能百分百证明真实传播。
- 收藏在抖音平台能力更完整，微信可能不支持。
- 添加桌面不是所有平台都支持，不支持时返回 `platform-unsupported`。
- 侧边栏应区分 `check` 和 `navigate`，一般 `check` 不发奖，`navigate` 才可能发奖。

推荐封装：

```ts
const ret = await H.reward.claimBySDKAction({
    rewardScene: 'shortcut_reward',
    rewardId: 'shortcut_once',
    action: () => H.sdk.addShortcut({
        rewardPolicy: 'completed',
    }),
});

if (ret.granted) {
    this.addDiamond(50);
}
```

## 15. 抖音录屏与视频分享

抖音小游戏增长能力建议补：

await H.sdk.startRecord({

const video = await H.sdk.stopRecord();

const ret = await H.sdk.shareVideo({

```ts
if (H.sdk.hasFeature('recordVideo')) {

```

## 16. 排行榜与好友能力

后续可抽象：

```ts
H.sdk.submitScore('level_rank', score);
H.sdk.showLeaderboard('level_rank');
H.sdk.inviteFriend({
    title: '一起挑战',
});
```

微信可能涉及开放数据域，抖音和 4399 也可能有各自排行榜能力。

原则仍然是：

- 平台差异放 Adapter。
- 业务只调用 `H.sdk`。
- 失败时统一返回 `platform-unsupported` 或 `failed`。

## 17. 埋点系统

### 17.1 平台控制台上报

框架内置 `HAnalyticsAdapter`，可选接入微信、抖音的自定义分析上报能力：

- 微信平台：运行时检测 `wx.reportAnalytics`。
- 抖音平台：运行时检测 `tt.reportAnalytics`。
- 其他平台：默认不上报，只保留本地缓存和自建后台 uploader。

默认不自动打开平台上报。项目需要在平台控制台先创建对应事件 ID，再在初始化时开启：

```ts
H.init({
    analytics: {
        platformReport: {
            enabled: true,
            allowEvents: [
                'ad_request',
                'ad_preload_start',
                'ad_preload_success',
                'ad_show',
                'ad_reward',
                'ad_cancel',
                'ad_no_fill',
                'ad_fail',
            ],
        },
    },
});
```

如果平台控制台事件 ID 和框架事件名不一致，可以使用 `eventNameMap`：

```ts
H.init({
    analytics: {
        platformReport: {
            enabled: true,
            eventNameMap: {
                ad_reward: 'ad_video_reward',
                ad_cancel: 'ad_video_cancel',
            },
        },
    },
});
```

平台上报只负责数据进入微信/抖音控制台，不替代广告平台自身的请求、展示、点击、收益统计。广告平台基础数据由微信/抖音广告 SDK 自动统计，框架上报的是业务漏斗事件和失败归因。

统一入口：

```ts
H.analytics.track('sdk_login_success', data);
H.analytics.track('ad_reward_success', data);
H.analytics.error('sdk_api_error', error);
```

## 18. 错误与日志系统

SDK 内部错误必须统一处理：

```ts
H.analytics.error('sdk_api_error', {
    action: 'addShortcut',
    reason: ret.reason,
    errorMessage: ret.errorMessage,
});
```

日志级别：

```ts
type HLogLevel = 'debug' | 'info' | 'warn' | 'error';
```

建议接口：

```ts
H.log.debug('message', data);
H.log.info('message', data);
H.log.warn('message', data);
H.log.error('message', data);
```

线上只上传 `warn/error`，本地开发可以打印 `debug/info`。

## 20. 初始化流程

推荐 Loading 阶段：

```ts
H.init({
    data: {
        namespace: 'GameName',
        modules: [
            { name: 'platformSession', defaultValue: {} },
            { name: 'reward', defaultValue: {} },
            { name: 'remoteConfig', defaultValue: {} },
        ],
    },
    platform: {
        platform: 'auto',
    },
    sdk: {
        sessionStorageModuleName: 'platformSession',
        actionCooldownMs: 30000,
    },
    ad: {
        platform: 'auto',
    },
});

await H.sdk.login();
await H.config.loadRemoteConfigInBackground();
```

初始化顺序：

```text
1. H.data 初始化本地存储。
2. H.platform 识别平台。
3. H.session 初始化会话时间。
4. H.ad 初始化广告配置。
5. H.sdk 初始化平台能力。
6. H.account 登录或恢复账号。
7. H.config 加载本地配置。
8. H.analytics 启动会话埋点。
9. Loading 继续加载资源。
```

const ret = await H.reward.claimBySDKAction({

## 22. 分阶段落地计划

### 第一阶段：SDK 基础增强

目标：让业务可以稳定调用，减少平台分支。

内容：

```text
1. H.config 配置中心。
2. H.sdk.hasFeature 能力检测。
3. H.reward 本地发奖系统。
4. H.analytics 本地日志和基础埋点。
```

完成标准：

- 业务不直接写广告 ID。
- 业务不直接判断平台。
- 分享、收藏、添加桌面都有 `rewardable`。
- 广告发奖走 `rewarded`。
- 所有奖励统一走 `H.reward`。

### 第二阶段：商业化稳定性

目标：提升广告稳定性和线上排障能力。

内容：

```text
1. 广告预加载。
2. 广告 ready 状态。
3. 广告错误归因。
4. 广告漏斗埋点。
5. 插屏、激励、Banner 全局频控。
```

完成标准：

- 能看到广告请求、展示、失败、发奖漏斗。
- 能区分 no-fill、配置缺失、平台不支持、频繁调用。
- 插屏不会在激励广告后短时间弹出。

### 第三阶段：增长能力

目标：补齐平台增长玩法。

内容：

```text
1. 分享奖励。
2. 收藏奖励。
3. 添加桌面奖励。
4. 侧边栏复访奖励。
5. 抖音录屏分享。
6. 排行榜和好友邀请预留。
```

完成标准：

- 所有增长行为都有能力检测。
- 不支持的平台自动隐藏入口或返回 unsupported。
- 发奖统一走 `H.reward`。

### 第四阶段：线上运营能力

目标：支持正式上线后的运营、灰度和风控。

内容：

```text
1. 远程配置。
2. 服务端登录。
3. 服务端发奖。
4. rewardToken 幂等。
5. 每日次数限制。
6. 风控数据上报。
```

完成标准：

- 线上可关闭任意广告位或奖励入口。
- 发奖不怕重复点击。
- 服务端可追踪每一笔奖励来源。

### 第五阶段：合规与账号体系

目标：满足长期运营和多渠道要求。

内容：

```text
1. 隐私协议。
2. 用户授权管理。
3. 游客账号。
4. token 刷新。
5. 实名和防沉迷预留。
6. 数据删除入口。
```

完成标准：

- 不合规状态不进入敏感能力。
- 日志和上报脱敏。
- 账号异常时可降级游客模式。

## 23. 优先开发清单

建议下一步按以下顺序实现：

```text
1. H.config
2. H.sdk.hasFeature
3. H.reward
4. H.analytics
5. 广告预加载和广告统计
6. 远程配置
7. H.account 服务端登录
8. 服务端发奖
9. 抖音录屏分享
10. 合规系统
```

最先落地的三件事：

```text
1. 配置中心：解决广告 ID、开关、冷却散落问题。
2. 能力检测：解决不同平台功能差异问题。
3. 统一发奖：解决奖励重复、判断混乱、业务散写问题。
```

## 24. 最终目标

最终业务代码应该变成：

```ts
if (!H.sdk.hasFeature('favorite')) {
    return;
}

const ret = await H.reward.claimBySDKAction({
    rewardScene: 'favorite_reward',
    rewardId: 'favorite_once',
    action: () => H.sdk.showFavoriteGuide({
        rewardPolicy: 'completed',
    }),
});

if (ret.granted) {
    this.addDiamond(50);
}
```

这时 SDK 才真正成为一套面向商业化、增长、运营和多渠道发布的高级小游戏框架，而不是简单的平台 API 包装。
