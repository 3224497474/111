# 战斗系统第二期规划

日期：2026-03-31

## 第二期目标

第二期不再停留在纯逻辑层，而是把第一期逻辑真正接进场景、表现和配置系统，形成“可看、可调、可扩展”的战斗版本。

核心目标：

1. 接入场景实体与输入
2. 接入角色动画与战斗表现
3. 完善技能效果系统
4. 补齐符纹更细的供能与联动
5. 补基础战斗 UI 与调试工具

## 第二期范围

### 2.1 场景接线

目标：

- 点击地面后把屏幕输入转换为战斗平面坐标
- 把 `BattleManager.setPlayerMoveTarget()` 接到实际场景控制器
- 战斗单位逻辑位置同步到场景节点位置

建议新增模块：

- `BattleSceneController`
- `BattleInputController`
- `BattleUnitView`
- `BattleProjectileView`

验收点：

- 玩家点击场景后，队伍真实移动
- 单位节点位置与逻辑位置保持同步
- 敌人追击、攻击、死亡能在场景中看到

### 2.2 动画与表现层

目标：

- 落地 `idle_01 / move_01 / attack_01 / death_01`
- 实现动画状态切换
- 接入投射物飞行、爆炸、近战刀光剑影

建议规则：

- `Idle -> Move`：收到移动指令
- `Move -> Attack`：目标进入攻击范围
- `Attack -> Idle`：攻击结束且无目标
- `Attack -> Move`：有移动目标
- `Any -> Death`：生命值为 `0`

建议新增模块：

- `BattleAnimationController`
- `BattleEffectController`
- `ProjectilePool`
- `EffectPool`

验收点：

- 玩家队伍和敌人都有完整基础动作
- 远程单位可见投射物飞行和命中爆炸
- 近战单位可见命中刀光/剑影

### 2.3 技能系统增强

目标：

- 从“伤害/治疗/控制”扩到更完整的效果系统
- 支持角色自带技能更丰富的表现与逻辑

建议补充能力：

- Buff / Debuff
- 持续伤害
- 护盾
- 加减速
- 硬控与软控区分
- 技能多段伤害
- 技能范围判定

建议新增模块：

- `BattleStatusSystem`
- `BattleSkillExecutor`
- `BattleFormulaService`

验收点：

- 至少支持 3 类不同风格自带技能
- 控制、治疗、伤害技能能稳定共存
- 技能命中与状态生效时机清晰

### 2.4 符纹联动增强

目标：

- 保持现有 `RuneBattleRuntime` 骨架不变
- 只扩展供能触发与联动表达力

建议新增触发类型：

- 普攻命中
- 普攻暴击
- 击杀目标
- 周期充能
- 释放自带技能后充能
- 生命低于阈值时充能

建议新增能力：

- 供能符纹限定只给某类技能符纹充能
- 技能符纹标签联动
- 终极符纹单独倍率或前置条件

验收点：

- 至少落地 2 个新供能触发
- 至少落地 1 套“供能符纹 + 技能符纹 + 终极符纹”联动样例

### 2.5 UI 与调试

目标：

- 让战斗状态可观测、可调试、可迭代

建议增加：

- 单位头顶血条、蓝条
- 技能释放提示
- 暴击/治疗飘字
- 调试面板：
  - 当前目标
  - 当前攻击冷却
  - 当前蓝量
  - 当前符纹充能

建议新增模块：

- `BattleHUDController`
- `BattleFloatingTextController`
- `BattleDebugPanel`

验收点：

- 设计和程序都能快速看懂当前战斗发生了什么
- 能直接观察到符纹充能与技能释放链路

## 第二期推荐实施顺序

建议按以下顺序推进：

1. 场景接线
2. 动画状态机
3. 投射物与近战特效资源池
4. 血条/蓝条/UI
5. 技能系统增强
6. 符纹供能增强
7. 调试工具和配置打磨

## 第二期产出物

第二期结束时，建议至少形成这些稳定成果：

- 可在场景中真实游玩的战斗原型
- 一套基础角色动画状态机
- 一套完整的单位视图层
- 一套可扩展的技能执行框架
- 一套更完整的符纹联动样例
- 一份新的第二期验收记录

## 风险点

第二期主要风险：

- 逻辑位置与场景节点位置同步抖动
- 动画切换和攻击命中帧不同步
- 投射物与技能效果重复结算
- 控制状态、移动状态、攻击状态互相覆盖
- 符纹技能、自带技能、普攻三条战斗链路耦合过深

建议原则：

- 逻辑层继续作为唯一真值
- View 层只做表现，不做伤害和状态判定
- 技能、符纹、普攻都统一走事件和结算接口

## 第二期完成标准

满足以下条件可判定第二期完成：

- 玩家点击地面后，真实场景中的整队角色可移动
- 敌我单位具备完整基础动作切换
- 普攻、投射物、近战特效、死亡表现都能看到
- 自带技能和符纹技能都能在场景中正常释放
- UI 能看到血量、蓝量、关键技能状态
- 至少补齐两类新的符纹供能触发
